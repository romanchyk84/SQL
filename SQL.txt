Main commands:
1.	DML (Data Manipulation Language)
	SELECT - Выбор из таблицы
	INSERT - Вставить в таблицу
	UPDATE - Обновить строку в таблице
	DELETE - Удалить строку в таблице
	MERGE - очень мощная команда, объединяет INSERT, UPDATE и DELETE
	
2.	DDL (Data Definition Language)
	CREATE - Создает новые таблицы
	ALTER - Изменяет структуру таблиц
	DROP - Удаляет объекты
	RENAME - Мзменяет название таблицы
	TRUNCATE - Удаляет все данные из таблицы, но при этом сама таблица остается
	
3.	TCL (Transaction Control Language)
	COMMIT - Подтверждает 
	ROLLBACK - Откатывает
	SAVEPOINT - точка сохранения
	
4.	DCL (Data Control Language) - Исключительно для работы Database администратора
	GRANT - Предоставляет кому-то определенные права (например для создания/редактирования/удаления данных)
	REVOKE - Удаляет, отбирает права пользователя
	
	
ТИПЫ ДАННЫХ:
	INTEGER - целые числа (например 5, 0, -200)
	NUMBER(p,s) - числовой тип данных, где p - это precision, это MAX количество цифр. S - scale, сколько от общего количества отводится на дробную часть.
		NUMBER(7,2) может иметь вид 12345,67 или 12345, но не может быть 123456,7. Целая часть может быть p-s цифр, например 7-2=5 цифр в целой части, в дробной только 2.
	CHAR(length) - фиксированная длинна симфолов. Недостоющее количество символов может заменяться пробелами.
		CHAR(8) может имет такой вид privet  , где 6 букв и в конце 2 пробела.
	VARCHAR2(size) - альфанумерик данные, переменная длина строки, в скобках указывается максимальная длина.
		VARCHAR2(8) - может имет вид из 6 букв, например privet, без заполнения недостающих символов пробелами. Экономит память.
	DATE - содержит информацию о дне, месяце, годе, часе, минуте и секунде
		12-Oct-22 23:15:18
	TIMESTAMP(6) - Также содержит информацию о дне, месяце, годе, часе, минуте и секунде, но также и содержит в себе долисекунды. В скобках размер долисекунды
		12-Oct-22 23:15:18.250000
	NULL - отсутствие данных, не содержит информации.
		0 или символ пробел - это не NULL, они занимают место в памяти, а NULL не занимает. Результат арифметических операций с NULL всегда будет NULL.
	
	DESCRIBE или DESC - выводит информацию о таблице, сколько в ней столбов, какие типы данных могут храниться. Например: DESCRIBE schema.tablename; или desc tablename;
	
	
	!!!Выражения с SELECT никогда не меняют данных!!!  SELECT - 3 фундаментальных концепции:
1.	PROJECTION - выбор тех столбцов, информация из которых нам интересна или важна
2.	SELECTION - выбор строк из таблицы по какому-то критерию, общему для чего-то
3.	JOINING - объединение таблиц, если нужно извлечь информацию из нескольких таблиц сразу
